北森三方平台集成
项目描述
  • 主要包含单点登录、消息待办集成、主数据集成三大功能模块
  • 单点登录：基于Oauth2.0协议搭建单点登录平台，对于常见的厂商（钉钉、泛微OA等）可通过Apollo配置零代码实现对接，且对非常见厂商提供租户组单点扩展能力
  • 消息待办集成：主要作用是打通北森与三方系统的信息屏障，通过把北森各个产品线（组织人事、招聘等）的消息、待办推送至三方系统（如企微、钉钉等），客户在不登录北森系统的情况下，也可以在三方系统中实时收到北森系统中的消息待办，并且可免登录（SSO）跳转到北森系统中处理相关待办。通过配置即可实现推送至内置的三方系统（如：企微、钉钉、泛微OA等），也可以低代码开发推送至任意客户系统。且可以根据客户的需求自定义接收标识、推送范围、产品线过滤等
  • 主数据同步：分为正向和反向主数据同步，把北森的人员信息、组织架构等同步至三方系统，实现一端维护，多端同步。标准功能支持企微、钉钉、泛微OA等存量客户较高的一些企业的主数据同步。针对一些存量较小，或者有自己OA系统的客户，提供了自研模块，低代码开发，快速满足客户的定制需求

停车场后台管理系统
涉及技术
  • 后台：Asp.Net MVC + MSSQLServer + EF + Unity→Asp.Net Core + MySql + EF + SqlSugar + Redis
  • 前台：EasyUI + JQuery + JavaScript + Ajax + Razor→Vue2.x + iviewUI + axios
项目描述
  • 该项目主要服务于停车场管理人员和现场的实施人员。为停车场管理人员提供权限管理、多维度车流订单报表、批量管理客户信息等；为现场的实现人员提供车场基础信息配置、多区域配置、硬件及计费、特殊收费配置等
负责内容
  • 负责整个项目的后台业务逻辑开发、前台页面的数据展示，并对项目进行架构优化
  • 根据角色实现按钮+动态列权限管理、利用NPOI实现客户信息的批量导入导出、使用Quartz.NET任务调度框架实现定时任务管理、基于MSDTC实现事务管理
  • 基于.net core对项目的现有业务进行重构，项目基于WebAPI设计，并采用前后端分离模式，划分为客户信息、用户信息、硬件配置、收费配置、车场信息、数据报表等模块。使用Swagger生成在线文档；采用MediatR实现进程内消息传递；使用Redis对热数据（例如出入口配置、收费配置、优惠券等）缓存；并采用读写分离的模式避免报表慢查询影响到收费系统；使用工作单元模式实现事务的统一管理。
  • 完成vue2.0从0到1架构实现，使用Vuex实现全局的状态管理，Vue-router实现对动态路由的管理
  • 三方平台数据对接，根据三方提供的文档编写接口、联调、并提供配置文档。例如：广州白云区公安系统、北京市西城区停车诱导系统平台、上海公安Agbox平台…………
解决的问题
  • 在几百万数据的情况下，MySql多表联查性能堪忧，采用反范式的数据库设计，冗余存储必要属性，减少联查提升性能
  • 在使用Redis做缓存时，发现需要经常性的序列化，影响性能，后采用hash结构，一个字段用于标记是否更改，一个字段存储缓存数据
  • 在使用axios发送请求时发生验签失败的问题，通过阅读源码发现axios会自动过滤掉值等于null或者underined的参数，可以通过重写paramsSerializer自定义序列化规则
